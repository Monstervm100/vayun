🧪 What is this page?

This page proves the dashboard isn't faking it. Each card below is an automatic test: it feeds the detection engine a known input and checks that the output is exactly what the project's requirements say it should be. Press "Run all runnable tests" (top right) or run cards one at a time, and watch them turn green (pass) or red (fail).

Two kinds of tests

Positive tests — "when an attack happens, does STARWATCH catch it?"
Negative tests — "when given garbage input or a broken server, does it fail gracefully instead of crashing?"

Positive test cases — does it catch real attacks?
Each test verifies one required behavior. The code like "FR2.1" links the test to the exact requirement in STARWATCH_SRD.md §4.
Negative test cases — does it handle bad input without crashing?
Invalid requests, server failures, and edge cases — the situations most likely to embarrass a live demo if untested.
